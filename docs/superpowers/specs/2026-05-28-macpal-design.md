# MacPal вҖ” Design Spec

**Date:** 2026-05-28
**Status:** Implemented (v0.1.0)
**Target:** macOS 14+, Swift 6.0, Xcode 16+

## Goal

A tiny always-on-top 2D desktop pet for macOS. Floats above every Space, walks the bottom of the screen, sleeps when idle, bounces when clicked, follows the cursor when dragged, and is controlled from a menu bar icon. No Dock icon, no docked window, no settings file.

## Non-Goals (YAGNI, v1)

- Multi-monitor edge climbing
- Skins / costumes / multiple pets
- Persistence of position or speed across launches
- Sound effects
- Sandboxing / notarization / App Store packaging
- AppleScript / URL scheme automation

All UI state lives in a single `@Observable PetController` running on `@MainActor`. AppKit owns the window and routes mouse events into the controller; SwiftUI draws everything inside the window.

## Components

| File | Responsibility |
| --- | --- |
| `MacPalApp.swift` | `@main App`. Hosts `MenuBarExtra` and `MenuBarContent`. Binds `AppDelegate` via `@NSApplicationDelegateAdaptor`. |
| `MenuBarContent` (in `MacPalApp.swift`) | Menu items: Wake/Sleep toggle, Speed submenu, Reset position, Quit. |
| `AppDelegate.swift` | `@MainActor` delegate. Sets activation policy to `.accessory` (no Dock icon), creates `PetWindow`, mounts `PetView` via `NSHostingView`, calls `controller.start()`. |
| `PetWindow.swift` | Borderless transparent `NSWindow`. `.floating` level. `canJoinAllSpaces` + `fullScreenAuxiliary` + `stationary`. Intercepts `mouseDown`/`mouseDragged`/`mouseUp` and forwards to the controller, distinguishing click (<4px move) from drag. |
| `PetState.swift` | `PetState` enum (`.idle`, `.walking(direction:)`, `.sleeping`, `.happy`, `.dragged`), `WalkDirection`, `PetSpeed`. |
| `PetController.swift` | `@MainActor @Observable` state machine. 30Hz `Timer` ticks behavior. Owns window-position math, idle-to-sleep timeout, click/drag handlers. |
| `PetView.swift` | SwiftUI body. Renders gradient blob + eyes + ears + mouth as `Shape`s. Animates blink and bob via local timers. Renders different mouth/posture per state. |


- Idle вҶ’ walking: 50% chance every 2вҖ“5s
- Walking вҶ’ idle: after 3вҖ“7s OR on screen edge (turns around, doesn't transition)
- Idle (30s, no user lock) вҶ’ sleeping
- Menu "Wake up" / "Sleep" toggles a `sleepLockedByUser` flag so the controller doesn't auto-flip back
- Click during sleep returns to sleep (locked)
- Drag: any state вҶ’ dragged вҶ’ idle on release

## Window Behavior

- Borderless, transparent, no shadow
- Floating level, joins all Spaces and full-screen Aux
- Initial position: bottom-center of `NSScreen.main.visibleFrame`, 40px above dock area
- Movement: `setFrameOrigin` driven by controller (walking) or by mouse delta (dragging)
- `canBecomeKey = false` so other apps keep keyboard focus
- App stays alive when window closes (`applicationShouldTerminateAfterLastWindowClosed = false`)

## Concurrency

Swift 6 strict concurrency is on. The model:

- `PetController` and `AppDelegate` are `@MainActor`
- Timer callbacks use `MainActor.assumeIsolated { вҖҰ }` to re-enter the main actor
- Delayed transitions use `Task { @MainActor in await Task.sleep(...) }` instead of `DispatchQueue.main.asyncAfter`

## Open Risks

- `mouseDown` in `PetWindow` fires across the full window bounds, including the transparent corners of the 96Г—96 frame. A circular hit test could shrink the interactive area but adds complexity вҖ” deferred.
- Walking math assumes the pet stays on the screen that contained it at launch. Moving the menu bar / disconnecting a display mid-session could put the pet off-screen. Reset Position recovers it.
